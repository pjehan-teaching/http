Présentation

Protocole HTTP

HTTP est un protocole permettant d'effectuer des requêtes sur un réseau et est principalement utilisé par les navigateur web pour récupérer les données des pages web.

Ce protocole a été créé par Tim Berners-Lee (également créateur du World Wide Web et du langage HTML) en 1989.

Protocole

Un protocole informatique est un ensemble de règles qui régissent les échanges de données ou le comportement collectif de processus ou d'ordinateurs en réseaux ou d'objets connectés.

Le modèle OSI

Le modèle OSI (Open Systems Interconnection) permet de décrire et standardiser les échanges de données entre les systèmes.

Son objectif est de permettre l'intéropérabilité des échanges en définissant des protocoles de communication standardisés.

Le modèle OSI

Ce modèle est découpé en 7 couches abstraites :

Application

Data

Permet d'afficher à l'utilisateur les données envoyées / réceptionnées.

  • HTTP
  • FTP
  • SMTP
  • SSH

Présentation

Data

Permet d'effectuer le codage/décodage des données.

  • SSL
  • SSH
  • FTP
  • JPEG

Session

Data

Permet d'ouvrir, fermer et gérer les sessions lors d'une communication.

  • RPC
  • RTCP
  • SCP
  • SOCKS

Transport

Segments

Permet de transférer les données sur le réseau.

  • TCP
  • UDP

Réseau

Packets

Permet de transférer des paquets entre les interfaces réseau (end-to-end communication).

  • IP
  • IPSec

Liaison

Frames

Permet de transférer des frames entre les interfaces réseau proches.

  • Ethernet
  • WiFi
  • Bluetooth

Physique

Bits

Composants physiques permettant la communication sur le réseau.

  • Câble RJ45
  • Fibre
  • WiFi

Visualiser une requête HTTP

traceroute est un programme permettant d'afficher le chemin parcouru par les paquets d'une requête sur un réseau.

traceroute google.com

Connaitre l'adresse IP d'un serveur

nslookup est un programme permettant d'afficher les adresses IP correspondant à un nom de domaine.

nslookup google.com

Requête / Réponse

Request : Récupérer le contenu à la racine du serveur répondant au domaine www.google.com

GET / HTTP/1.1
Host: www.google.com

Response : Le serveur répond "OK" (données retournées sans erreur) et précise que la réponse contient du code HTML

HTTP/1.1 200 OK
Content-Type: text/html
